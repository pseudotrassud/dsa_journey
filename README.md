# About

This repository is more than a collection of LeetCode solutions. It is a personal knowledge base organized around **algorithmic patterns**—the building blocks that recur across hundreds of coding interview problems.

Each topic folder will gradually evolve to include:

* Clean Java implementations
* Pattern explanations
* Reusable templates
* Complexity analysis
* Interview observations
* Common pitfalls and edge cases

The repository is intended to grow continuously as I learn new topics and revisit existing ones with deeper understanding.

# Why Organize by Patterns?

Organizing problems by difficulty or problem number often creates the illusion of progress without improving transferability.

Instead, this repository groups problems by **algorithmic patterns**.

Mastering a pattern such as **Sliding Window** or **Two Pointers** means learning to recognize a family of problems and applying a consistent strategy across different variations. This approach builds intuition, improves pattern recognition, and better reflects how coding interviews are approached in practice.
